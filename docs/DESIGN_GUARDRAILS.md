# DESIGN GUARDRAILS

## Product Identity

Inside the System is an auditable process simulation / guided control-board RPG.

It is not a dashboard. It is not an arcade game. It is not shallow gamification.

## Core Design Rule

Every screen element must support at least one of these:

- decision
- reveal
- consequence
- evidence
- control
- ownership
- risk
- recovery
- audit trail
- AAR learning

## Guidance Rule

Guidance explains tradeoffs. It does not tell the player the exact answer.

## Hidden State Rule

Hidden information must be discoverable through inspection, evidence review, validation, or control testing.

## Safety / Scope Rule

Do not model classified, sensitive, adversary, targeting, or real-world procedural vulnerabilities.
